## Summary
Split get_result (text-only JSON) and get_artifacts (images per chunk) for token efficiency. get_result now returns lightweight summaries + artifact metadata without base64 images. New get_artifacts tool fetches images for a specific chunk by index, enabling on-demand deep dive. Reduces initial result fetch from ~25K to ~5-7K tokens.

## Changed areas
- internal/handler/mcp/get_result.go — removed image embedding, text-only JSON
- internal/handler/mcp/get_artifacts.go — new tool for chunk-specific images
- internal/handler/mcp/handler.go — registered get_artifacts
- 6 MCP tools total: add_video, get_status, get_result, get_artifacts, delete_task, retry_task
