## Summary
Added pipeline checkpoint system that saves state to `checkpoint.json` after each step, enabling failed tasks to resume from the last completed step without re-downloading or re-processing. Added `retry_task` MCP tool to re-enqueue failed tasks. Moved extracted frames to a `frames/` subdirectory. Updated Ollama config to use `host.docker.internal` with increased timeout for model loading.

## Changed areas
- `internal/usecase/pipeline/state.go` — added JSON serialization, `Save()` and `LoadCheckpoint()` methods
- `internal/usecase/pipeline/runner.go` — checkpoint save after each step, resume logic on retry
- `internal/handler/mcp/retry_task.go` — new `retry_task` MCP tool handler
- `internal/handler/mcp/deps.go` — added `Retry` to `TaskService` interface
- `internal/handler/mcp/handler.go` — registered `retry_task` tool
